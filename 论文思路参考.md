# 摘要  
本研究针对社交媒体信息级联传播（如微博、Twitter 等平台上的消息转发）任务，提出基于动态图神经网络（DyGNN）的**级联规模预测**与**关键传播模式解释**方案。具体而言，我们将每条级联的早期传播过程划分为离散时间片，构建随时间变化的传播子图，并提取节点级别和图级别特征。模型方面，利用图注意力网络（GAT）对每个时间片的传播结构进行编码，然后通过时间序列模型（如 GRU + 注意力机制）学习级联演化表示，最终预测传播规模 \(Y=\log(1+N)\)。解释方面，我们为每个时间片设计可学习的时空**掩码** \(M_k\)，提取出对预测结果贡献最大的传播边或子结构，并通过反事实删除实验和稳定性检验评估其有效性。参考前沿研究（如 ICLR 2024 年的DyGNN因果解释模型【38†L22-L31】、动态图扩散-变分网络【37†L63-L70】、FedDGL级联预测方法【35†L71-L74】），我们的方案在保证预测精度的同时，能够提供人类可解释的传播模式见解。实验计划采用真实微博/推特数据集（如 Weibo/Twitter CTCP 数据集【35†L157-L160】），进行系统的训练验证和对比分析，并通过表格和可视化（如流程图、示例网络图）呈现关键结果。创新点包括：动态图建模与静态方法的对比、时空掩码解释机制以及因果启发式的解释验证。总体来说，本方案具有明确任务定位、丰富的方法细节和完善的验证设计，切合大学生建模竞赛的要求和创新指标。

## 1. 研究背景与目标  
信息级联传播（cascade）是社交媒体上消息传播的关键现象，其最终规模（转发数）反映了信息的热度与影响力。预测级联规模在社会网络分析、公共舆论监控和风险防范中具有重要应用。传统方法主要基于自激点过程或静态图模型进行预测，难以捕捉信息传播的动态演化规律【35†L61-L69】。近年来，动态图神经网络（DyGNN）在动态网络预测中取得显著成果，但通常缺乏可解释性【38†L60-L69】【38†L22-L31】。本项目旨在解决以下问题：**1.** 如何有效表示和学习信息级联的时序演化过程以提升预测精度；**2.** 在保持高精度的同时，如何识别和解释模型预测依赖的关键传播模式（局部子图或时间窗口）；**3.** 如何验证所提解释的有效性和稳定性。针对这些目标，我们将建立一套完整的研究框架，包括数据预处理、DyGNN模型构建、可解释性模块设计与实验验证。预期成果是在竞赛时间内得到**技术路线清晰**、**结果可解释**的研究方案。

## 2. 数据来源与级联样本构建  
- **数据来源**：我们选取真实社交网络数据集，例如公开的微博、Twitter 级联数据【35†L157-L160】。典型数据集包括 CTCP（Chinese Twitter Cascade Prediction）GitHub仓库提供的微博/推特级联数据【35†L157-L160】。数据应包含每条消息的发布者、发布时间，以及随后所有转发（或评论）的用户与时间戳信息。  
- **级联样本构建**：针对每个原始信息（源节点），将其后续的全部转发行为视为级联传播样本 \(\mathcal{C}=\{(u_i,t_i)\}_{i=1}^N\)，其中 \(N\) 为最终传播规模（转发总数），用户 \(u_i\) 在时间 \(t_i\) 时转发该信息。定义观测窗口为 \([0,T]\)，我们只利用前期传播 \(\mathcal{C}^{(T)}=\{(u_i,t_i)\mid t_i\le T\}\) 来预测最终 \(N\)（预测目标取对数形式 \(Y=\log(1+N)\)）。  
- **时间切分**：将观测窗口 \([0,T]\) 划分为 \(K\) 个时间片 \([\tau_{k-1},\tau_k)\)。例如，可根据任务需求设定固定时长（如5分钟或1小时）为时间片长度【35†L71-L74】。  
- **节点与边特征**：对于第 \(k\) 个时间片，定义传播子图 \(G_k=(V_k,E_k)\)，其中 \(V_k\) 为截至 \(\tau_k\) 时所有参与传播的用户集合，\(E_k\) 为相应的转发边集合（若用户 \(i\) 在该片期被用户 \(j\) 转发，则存在有向边 \(j\to i\)）。我们为每个节点 \(v_i\in V_k\) 提取特征向量 \(\mathbf{x}_i^{(k)}\)，可包括：用户属性（粉丝数、关注数、是否认证等）、行为属性（过去转发数、活跃度）、时间属性（首次转发延迟）、内容属性（情感倾向或主题 embedding）等【35†L71-L74】。同时，对每个时间片构造**宏观特征** \(\mathbf{m}_k\)，如当前级联规模 \(n_k=|V_k|\)、增长率、传播深度/宽度等统计量【35†L71-L74】。FedDGL方法便采用类似思路：“将观测时间分段，提取微观（节点级）和宏观（图级）特征”【35†L71-L74】。

## 3. DyGNN 级联规模预测模型  

1. **动态图构建**：将上述级联样本表示为动态图序列 \(\mathcal{G}^{(T)}=\{G_1,\dots,G_K\}\)，其中每个 \(G_k=(V_k,E_k,X_k,M_k)\) 包括节点集、边集、节点特征矩阵 \(X_k=[\mathbf{x}_i^{(k)}]_{i\in V_k}\) 和宏观特征向量 \(\mathbf{m}_k\)。  
2. **图结构编码（GAT）**：对每个 \(G_k\) 内的节点进行图卷积编码，采用图注意力机制公式【38†L22-L31】：  
   \[
   \mathbf{h}_i^{(k)}
   =\sigma\Bigl(\sum_{j\in \mathcal{N}_k(i)}\alpha_{ij}^{(k)} W \,\mathbf{x}_j^{(k)}\Bigr),
   \quad
   \alpha_{ij}^{(k)}=\frac{\exp(\text{LeakyReLU}(\mathbf{a}^\top[W\mathbf{x}_i^{(k)}\Vert W\mathbf{x}_j^{(k)}]))}{\sum_{l\in\mathcal{N}_k(i)}\exp(\dots)},
   \]
   其中 \(\mathbf{h}_i^{(k)}\) 为节点 \(i\) 在第 \(k\) 时间片的隐表示，\(W\)、\(\mathbf{a}\) 为可学习参数，\(\sigma(\cdot)\) 为激活函数。【38†L22-L31】  
3. **图级表示与融合**：对每个时间片的节点表示进行归纳读出（如求平均）：  
   \[
     \mathbf{g}_k = \frac{1}{|V_k|}\sum_{i\in V_k}\mathbf{h}_i^{(k)}.
   \]
   将其与宏观特征 \(\mathbf{m}_k\) 拼接后线性变换得到时片整体表示：  
   \(\mathbf{c}_k = \phi\bigl(W_c[\mathbf{g}_k\Vert\mathbf{m}_k]+\mathbf{b}_c\bigr)\)。  
4. **时序演化建模**：将 \(\{\mathbf{c}_k\}_{k=1}^K\) 输入时间序列模型（如 GRU 或双向 LSTM）。以 GRU 为例：  
   \[
     \mathbf{s}_k = \text{GRU}(\mathbf{c}_k,\mathbf{s}_{k-1}),\quad k=1,\dots,K.
   \]
   另外，为关注各片期的重要性，可引入时间注意力权重 \(\beta_k = \frac{\exp(\mathbf{q}^\top \mathbf{s}_k)}{\sum_r \exp(\mathbf{q}^\top \mathbf{s}_r)}\)，并聚合得到最终传播表示  
   \(\mathbf{z} = \sum_{k=1}^K \beta_k \mathbf{s}_k\)。  
5. **级联规模预测**：使用动态表示 \(\mathbf{z}\) 进行线性回归预测：  
   \(\hat{Y} = W_y \mathbf{z} + b_y\), 对应真实目标 \(Y=\log(1+N)\)。训练采用均方误差损失：  
   \[
     L_{\text{pred}} = \frac{1}{M}\sum_{m=1}^M (Y_m - \hat{Y}_m)^2,
   \]
   其中 \(M\) 为样本数量。通过比较不同方法，我们可证明 DyGNN 对比传统静态方法（如仅基于初始图的 GCN）或浅层统计模型有更好的预测性能【35†L71-L79】。

## 4. 关键传播模式解释模块  
为了解释模型预测结果，我们设计**时空掩码**来识别重要传播结构。具体步骤如下：  

- **掩码定义**：令 \(M_k\) 表示时间片 \(k\) 上节点或边的重要性掩码矩阵（元素取值 \(0\sim1\)）。例如可对传播子图的边加权：\(M_k(i,j)\in[0,1]\) 表示边 \((i,j)\) 的重要程度。【38†L22-L31】文章采用类似的“因果掩码”来捕获关键边。  
- **关键子图提取**：根据阈值 \(\delta\)，从原图 \(G_k\) 中提取解释子图 \(\tilde{G}_k=(V_k,\tilde{E}_k)\)，其中 \(\tilde{E}_k=\{(i,j)\in E_k\mid M_k(i,j)>\delta\}\)。整个关键传播模式表示为 \(\tilde{\mathcal{G}}^{(T)}=\{\tilde{G}_1,\dots,\tilde{G}_K\}\)。  
- **解释损失**：期望在保留尽量少的关键结构前提下使预测结果不变。定义带掩码预测为 \(\hat{Y}_{\mathcal{M}}\)，则解释优化目标为：  
  \[
    L_{\text{exp}}
    = (\hat{Y}-\hat{Y}_{\mathcal{M}})^2 
    + \lambda_1\sum_{k=1}^K \|M_k\|_1 
    + \lambda_2\sum_{k=2}^K \|M_k - M_{k-1}\|^2,
  \]
  其中第一项保证保真度（预测变化尽量小），第二项促使掩码稀疏（重要结构少），第三项平滑相邻时间片的掩码变化（提高解释时序稳定性）。公式灵感来源于图解释器方法【38†L22-L31】。  
- **反事实贡献**：对于样本 \(c\)，令原始预测值 \(\hat{Y}\)，去掉关键子图后重新预测得到 \(\hat{Y}^{(-\mathcal{M})}\)。定义关键模式的**反事实贡献**为 \(\Delta = \hat{Y}-\hat{Y}^{(-\mathcal{M})}\)。若 \(\Delta\) 显著大于零，则表明移除该模式会大幅降低预测结果，说明该模式对预测至关重要。这种度量借鉴了因果推断的反事实思想，但并未进行严格的因果效应计算，仅作为解释质量的启发性指标。【38†L22-L31】  
- **统计指标**：  
  - **Drop**：删除关键模式后的预测下降量 \( \text{Drop} = \hat{Y}-\hat{Y}_{\mathcal{M}} \)。Drop 越大，说明该模式承载了更多预测信息。  
  - **Stability**：对同一样本进行多次解释（不同初始化或不同扰动），得到两组掩码 \(M^{(a)},M^{(b)}\)，定义稳定性指标  
    \(\text{Stability}=1-\frac{\|M^{(a)}-M^{(b)}\|_1}{\|M^{(a)}\|_1+\|M^{(b)}\|_1}\)，  
    值越接近 1 表示解释结果越一致。  

## 5. 验证指标与实验方案  
- **预测性能指标**：采用均方误差（MSE）和平均绝对误差（MAE）评估回归准确度：  
  \[
    \text{MSE}=\frac{1}{M}\sum_{m=1}^M(Y_m-\hat{Y}_m)^2,\quad
    \text{MAE}=\frac{1}{M}\sum_{m=1}^M|Y_m-\hat{Y}_m|.
  \]
  如果任务扩展为判别级联是否成为“热点”类别，则可使用分类指标如 AUC、F1 分数等。  
- **数据划分与实验流程**：将数据按时间或样本随机分为训练集、验证集和测试集（如 8∶1∶1）。在训练集中进行模型训练和超参数调优，验证集中选择模型，最后在测试集上报告性能。可采用交叉验证确保结果稳健。  
- **基线模型**：对比实验可包括：**传统统计模型**（如线性回归、岭回归等）、**机器学习模型**（如随机森林、XGBoost）、**静态图神经网络**（如只用原始扩散图的 GCN）以及已有的动态图方法（例如 FedDGL【35†L71-L74】）。此类基线帮助量化动态图方法的提升。  
- **消融与删除实验**：验证关键传播模式的有效性：在测试集上分别去除说明子图（置零对应结构）或随机扰动图结构，观察预测误差的变化（Drop 值）。若删除重要子结构误差显著上升，说明解释有效。  
- **跨事件稳定性检验**：分析不同事件类别或时间段模型解释结果的一致性，以检验方法对不同场景的泛化能力。  
- **可视化与表格**：通过表格展示不同模型的预测性能对比，以及不同评估指标（如 MSE、MAE、AUC）在各模型下的数值。可视化示例如 **图1** 中级联示例图和预测结果与掩码示例图，使结果直观。

## 6. 实施细节与资源需求  
- **开发工具**：使用 Python 平台，图神经网络可采用 PyTorch Geometric 或 DGL 等库实现 GAT、GRU 等模块。解释模块涉及可学习掩码，可通过类似 GNNExplainer 的优化手段实现。【38†L22-L31】  
- **超参数调节**：需要调节的主要参数包括：GAT 层数与头数、隐藏维度、GRU 隐藏单元数、学习率、L2 正则化系数、解释损失权重 \(\lambda_{1,2}\) 和整体平衡系数 \(\gamma\)。可通过验证集网格搜索或贝叶斯优化确定。  
- **计算资源**：数据预处理和模型训练建议使用 GPU 服务器（示例：NVIDIA Tesla V100 以上），以应对图卷积和序列模型的计算需求。级联数据通常规模适中，应可在数小时内训练完成。  
- **时间计划（甘特图示例）**：  

  | 周次        | 主要任务                    |
  |------------|----------------------------|
  | 第1–2周    | 数据采集与清洗；构建动态图     |
  | 第3–4周    | 实现并训练 DyGNN 预测模型      |
  | 第5–6周    | 设计解释模块；掩码学习与优化   |
  | 第7周      | 实验调试；性能评估            |
  | 第8周      | 结果分析；报告撰写与答辩准备    |

- **风险及对策**：可能面临的问题包括数据稀疏或不平衡（可通过采样或正则化处理），模型过拟合（使用 dropout、早停等策略）。解释模块的优化可能不易收敛，需要调节掩码初始化和正则强度。预先设计少量案例并进行参数敏感性分析可降低风险。

## 7. 创新点及可行性论证  
- **动静结合的特征建模**：利用动态图神经网络同时挖掘节点级（微观）和图级（宏观）传播特征，相比仅基于静态图的方法能更全面捕捉级联演化规律【35†L71-L74】。  
- **时空关键模式解释**：提出在每个时间片上使用可学习掩码提取关键传播结构，不仅解释“哪些用户重要”，更重视“哪些传播路径/子图重要”，提升解释的时序性和过程性【38†L22-L31】。  
- **因果启发式验证**：引入反事实删除实验和掩码平滑约束，借鉴因果推断思想降低伪相关干扰，增强解释的可信度。这在动态图可解释研究中属于较新尝试【38†L22-L31】【37†L63-L70】。  
- **体系完整、可行性高**：数据来源充足（已有开源数据集【35†L157-L160】）、方法可借鉴现有GNN工具快速开发、实验设计完整。创新点聚焦于模型解释层面，易与现有预测研究区别开来，符合竞赛要求。
